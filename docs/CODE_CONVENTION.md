# CODE_CONVENTION.md — Clean Code 원칙

`DummyDataGenerator` 모듈 개발 시 아래 로버트 C. 마틴(Robert C. Martin)의 클린 코드 원칙을 따른다.

## 원칙

- 읽기 쉬운 코드가 좋은 코드이다. 가독성이 최우선이다.
- 이해하기 쉬운 코드는 곧 유지보수가 좋은 코드이다.
- 이름에는 의미가 있어야 하며, 함수는 한 가지 역할을 담당해야 한다.
- 코드도 글처럼 읽히도록 해야 하며, 하나의 문서처럼 취급해야 한다.
- 클래스는 하나의 책임을 갖도록 한다. (SRP, 단일 책임 원칙)
- 테스트 코드는 필수이며, 자주 발생되는 중복은 제거해야 한다.
